# 题目7：有符号4位二进制数的显示与取反/补码运算（中断方式 + 数码管扫描）

## 题目要求
- **开关中断自动读取**：最左边4个数码管始终显示最右边4位独立开关的**二进制值**（每位显示 0 或 1）
- **BTNC（bit0）**：最右边2个数码管显示该二进制码对应的**有符号十进制值**（符号位为 bit3，数值为 bit2~bit0）
- **BTNL（bit2）**：每按一次，在最右边2个数码管交替显示：
  - 第一次：**取反后的十进制值**（符号位不取反，仅低3位取反）
  - 第二次：**原值**（同 BTNC）
  - 第三次：取反值… 如此循环

## 硬件资源
| 外设 | GPIO | 方向 |
|------|------|------|
| 独立开关(16位) | GPIO0_CH1 | 输入 |
| LED灯(16位) | GPIO0_CH2 | 输出 |
| 数码管位选(8位) | GPIO1_CH1 | 输出 |
| 数码管段码(8位) | GPIO1_CH2 | 输出 |
| 独立按键(5位) | GPIO2_CH1 | 输入 |

## 中断架构
| 中断源 | 掩码 | 作用 |
|--------|------|------|
| GPIO0（开关） | 0x1 | 检测开关值变化，更新左边4位二进制显示 |
| GPIO2（按键） | 0x2 | 检测 BTNC/BTNL/BTNR 按键按下 |
| TIMER0 T0 | 0x4 | 数码管动态扫描（约10μs） |

## 核心实现逻辑

### 1. 开关中断 —— 左边4位显示二进制
```c
void switch_handle(void)
{
    char sw = Xil_In16(...);       // 读取16位开关值
    char low4 = sw & 0xF;          // 取最右边4位

    for (int i = 0; i < 4; i++)
    {
        if (low4 & (0x8 >> i))     // 从高位到低位逐位判断
            segcode[i] = 0xf9;     // 显示 "1"
        else
            segcode[i] = 0xc0;     // 显示 "0"
    }
}
```

### 2. BTNC —— 显示有符号十进制值
```c
char sign = low4 & 0x8;       // 符号位
char number = low4 & 0x7;     // 数值部分（bit2~bit0）

if (sign) {                    // 负数
    segcode[6] = 0xbf;        // 显示负号 "-"
    segcode[7] = segtable_hex[number];  // 显示数值
} else {                       // 正数
    segcode[6] = 0xff;        // 熄灭负号位
    segcode[7] = segtable_hex[number];
}
```

### 3. BTNL —— 交替显示取反值 / 原值
```c
char number_inverted = (~low4) & 0x7;  // 仅低3位取反，符号位不取反

if (count == 0) {
    /* 显示取反后的十进制值 */
    segcode[7] = segtable_hex[number_inverted];
    if (sign) segcode[6] = 0xbf;  // 符号位保留
    else      segcode[6] = 0xff;
    count++;
} else if (count == 1) {
    /* 显示原值（同 BTNC） */
    segcode[7] = segtable_hex[number];
    if (sign) segcode[6] = 0xbf;
    else      segcode[6] = 0xff;
    count = 0;
}
```

## 运算示例对照

| 开关值(bin) | 有符号数 | BTNC显示 | BTNL 第一次 | BTNL 第二次 |
|-------------|---------|----------|------------|------------|
| 0110 | +6 | `[6]` | 取反低3位: 001→**1** → `[1]` | 恢复: **6** → `[6]` |
| 1100 | -4 | `[-4]` | 取反低3位: 100→011=3 → `[-3]` | 恢复: **4** → `[-4]` |
| 0101 | +5 | `[5]` | 取反低3位: 101→010=2 → `[2]` | 恢复: **5** → `[5]` |
| 1011 | -3 | `[-3]` | 取反低3位: 011→100=4 → `[-4]` | 恢复: **3** → `[-3]` |

### 取反细节说明（以 1100 为例）
```
原始:  1100  (bit3=1 负数，数值 100=4 → -4)
取反低3位: 1 100 → 1 011 (数值部分 100→011=3 → -3)
所以 count=0 显示 -3，count=1 恢复显示 -4
```

### 取反细节说明（以 0110 为例）
```
原始:  0110  (bit3=0 正数，数值 110=6 → +6)
取反低3位: 0 110 → 0 001 (数值部分 110→001=1 → +1)
所以 count=0 显示 1，count=1 恢复显示 6
```

## 数码管显示布局
```
  segcode[0] [1] [2] [3]    [4] [5] [6]  [7]
  ┌────┬────┬────┬────┐  ┌────┬────┬────┬────┐
  │  1 │  1 │  0 │  0 │  │  灭│  灭│  - │  4 │
  └────┴────┴────┴────┘  └────┴────┴────┴────┘
  左4位：二进制显示          右4位：BTNC/BTNL结果
  (开关=1100=-4)             (显示 -4)
```

## 关键点
- **开关中断（GPIO0）仅当开关值变化时触发**，上电初始值不会自动触发。需要确保 `Initialization()` 最后使能中断前已调用 `switch_handle()` 或做了一次手动读取
- **符号位（bit3）不参与取反**：取反运算 `(~low4) & 0x7` 只对低3位操作，高位置0，保证符号位不变
- **count 变量**在 BTNL 中交替 0→1→0→1…，控制显示取反值还是原值
- 负号段码 `0xbf` 对应共阳极数码管显示 `-`（仅 g 段不亮）
- 数码管扫描由 T0 定时器驱动，约 10μs 切换一位，8位循环，人眼看起来稳定显示

## 可能存在的问题（已修复）
- **断点问题**：原代码 `switch_handle()` 中 `char sw = Xil_In16(...)` 用了 `char`（8位）存16位值，虽然取低4位不影响结果，但不规范。建议改为 `unsigned short`
- **上电无显示**：GPIO0 中断只会在开关值**变化**时触发。若上电后用户不动开关，`switch_handle()` 不执行，左边4位 segcode[0~3]=0xff 全灭。需在初始化末尾手动调用一次
